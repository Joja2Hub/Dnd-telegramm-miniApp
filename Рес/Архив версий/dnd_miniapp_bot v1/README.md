# D&D Telegram Mini App

Готовый MVP Telegram Mini App для D&D-кампаний.

Версия v6: исправлен выбор цвета персонажа — теперь цвет выбирается готовыми кнопками с явным подтверждением, а код приглашения показан отдельным заметным блоком.

В проекте есть:

- Telegram-бот для `/start`, `/join`, кнопки открытия Mini App и уведомлений.
- Backend API на FastAPI.
- SQLite-база данных.
- Mini App-интерфейс для мастера и игроков.
- Фэнтези-система HP, КД, боли, травм и брони.
- Генераторы настроения, погоды и событий отдельным разделом.

## Что умеет мастер

В Mini App мастер может:

- создавать кампании;
- при создании выбирать модификаторы:
  - травмы;
  - броня;
- добавлять персонажей;
- видеть карточки всех игроков;
- наносить обычную атаку без проверки КД: если не попал, просто не применяешь урон;
- наносить бронебойный урон, если включена броня;
- наносить массовый урон нескольким целям;
- вручную менять:
  - текущее HP;
  - максимум HP;
  - штраф максимального HP;
  - КД;
  - боль;
  - текущую броню;
  - максимум брони;
  - износ брони;
- применять полное излечение;
- чинить броню по броску d20;
- добавлять и удалять статусы;
- подтверждать заявки игроков;
- запускать генераторы;
- отправлять результат генератора всем привязанным игрокам.

## Что умеет игрок

Игрок в Mini App может:

- видеть свою карточку персонажа;
- видеть HP, броню, КД, боль, травмы и статусы;
- запросить лечение;
- запросить стабилизацию травмы;
- запросить ремонт брони с вводом результата d20.

Игрок не может сам менять HP, броню, травмы или статусы.


## Код приглашения

У мастера код приглашения теперь показывается отдельным заметным блоком наверху кампании. Его можно скопировать кнопкой.

Игрок подключается так:

```text
/join КОД
```

Например:

```text
/join ABC123
```

После этого игрок выбирает свободного персонажа.

## Как работает броня

Если в кампании включена броня, у персонажа есть:

- текущая броня;
- максимальная броня;
- износ брони.

Обычный урон сначала снимает броню. Если урон больше текущей брони, остаток проходит в HP.

Пример:

```text
HP: 50
Броня: 20
Урон: 25

Броня: 20 → 0
HP: 50 → 45
```

Бронебойный урон наносит полный урон персонажу и одновременно повреждает броню.

Пример:

```text
HP: 50
Броня: 20
Бронебойный урон: 10

Броня: 20 → 10
HP: 50 → 40
```

## Как работает ремонт брони

Игрок или мастер вводит результат d20.

- ровно `20` — критический успех, максимум брони не снижается;
- больше `20` не считается критом;
- при ремонте броня восстанавливается до нового максимума;
- максимум брони снижается на 1–5, кроме натуральной 20.

Таблица износа:

```text
20      → -0 максимума брони
17–19   → -1
13–16   → -2
9–12    → -3
5–8     → -4
1–4     → -5
```

## Как работают травмы

Если в кампании включены травмы, урон может вызвать травму. Травмы уменьшают максимальное HP не статическим числом, а процентом от базового максимального HP персонажа.

Примерные значения:

```text
Лёгкая травма: 2–5% от базового максимального HP
Средняя травма: 8–13%
Тяжёлая травма: 15–24%
```

Крупный урон может немного усилить потерю максимального HP.

## Запуск локально без Docker

### 1. Создай бота

В Telegram открой `@BotFather` и создай бота:

```text
/newbot
```

Скопируй токен.

### 2. Создай `.env`

Скопируй файл:

```bash
cp .env.example .env
```

На Windows можно просто скопировать `.env.example` и переименовать копию в `.env`.

Заполни:

```env
BOT_TOKEN=твой_токен_от_BotFather
BASE_URL=https://твой-публичный-адрес
DB_PATH=data/dnd_miniapp.sqlite3
HOST=0.0.0.0
PORT=8000
DEV_MODE=1
```

Для полноценного Mini App нужен HTTPS-адрес. Для локального теста удобно использовать ngrok.

### 3. Установи зависимости

```bash
python -m venv .venv
```

Linux/macOS:

```bash
source .venv/bin/activate
```

Windows:

```bat
.venv\Scripts\activate
```

Установка:

```bash
pip install -r requirements.txt
```

### 4. Запусти проект

```bash
python main.py
```

После запуска backend будет доступен на:

```text
http://localhost:8000
```


5. Перезапусти `python main.py`.

```text

```

## Dev-режим в обычном браузере

Если открыть `http://localhost:8000` не из Telegram, приложение попросит ввести Telegram ID.

Это работает только при:

```env
DEV_MODE=1
```

Так можно быстро тестировать интерфейс мастера и игрока без Telegram.

На реальном сервере лучше поставить:

```env
DEV_MODE=0
```

Тогда доступ будет только через Telegram Mini App с проверкой `initData`.

## Важные команды бота

```text
/start  — приветствие и кнопка открытия Mini App
/app    — кнопка открытия Mini App
/join КОД — подключиться к кампании как игрок
```

## Примечания

- Mini App должен открываться по HTTPS-адресу.
- Данные хранятся в SQLite-файле из `DB_PATH`.
- Если используешь Docker, база лежит в папке `data/` благодаря volume.
- Бот и backend запускаются одним процессом через `main.py`.
- Если `BOT_TOKEN` не задан, запустится только web/API, но уведомления и команды Telegram работать не будут.

В Mini App добавлены новые удобства для игры:

- игрок при заявке лечения указывает конкретное количество HP;

- числовые поля очищаются при первом фокусе, чтобы не получалось `1215` вместо `12`;

- при нажатии на боль открывается справка о текущем болевом эффекте;
- генератор настроения больше не показывает консольные теги `GOOD/social`, вместо этого используются русские подписи, цвета и эмодзи;
- в журнале мастера появилась кнопка отмены последнего действия.

### Отмена последнего действия

Кнопка в журнале откатывает последнее действие, для которого был сохранён снимок состояния. Сейчас поддерживаются откаты урона, ручных правок, лечения, ремонта брони, статусов, создания персонажа и подтверждения заявок. Генераторы и отправка сообщений игрокам не откатываются, потому что они не меняют состояние персонажей.

### Важно про `.env`

Файл `.env` с токеном бота не кладётся в архив специально. Оставь свой текущий `.env` рядом с проектом или скопируй его из предыдущей папки. Не публикуй `.env`, потому что там хранится токен Telegram-бота.

## Обновление v10 — контроллер боя и инициатива

В Mini App добавлен отдельный инструмент для менеджмента боя.

### Для мастера

Во вкладке **Бой** теперь есть кнопка **Начать бой**. После запуска:

1. Все персонажи кампании автоматически попадают в инициативу.
2. Игроки вводят инициативу в своей вкладке **Бой**.
3. Мастер добавляет врагов через кнопку **Добавить врага**.
4. У врага задаются:
   - имя;
   - HP;
   - КД;
   - инициатива;
   - цвет;
   - описание для игроков;
   - скрывать ли состояние здоровья.
5. Когда все готовы, мастер нажимает **Начать порядок ходов**.
6. Текущий ход отображается в карусели инициативы.
7. Мастер может:
   - передать ход;
   - нанести урон врагу;
   - изменить данные врага;
   - удалить участника из инициативы;
   - менять инициативу участника, чтобы перестроить порядок ходов;
   - добавить нового врага во время боя;
   - закончить бой.

Если HP врага падает до 0, он автоматически убирается из активной инициативы.

### Для игроков

У игроков появилась вкладка **Бой**. В ней они видят:

- поле ввода своей инициативы, пока бой в стадии подготовки;
- порядок ходов;
- текущего участника хода;
- список врагов в закрытом виде.

При нажатии на врага игрок видит только публичную информацию:

- имя;
- цвет;
- описание от мастера;
- примерное состояние, если мастер не включил скрытие HP.

Точные HP и КД врагов доступны только мастеру.

## v15 — достижения и библиотека рамок

Добавлена система достижений, привязанная к Telegram ID игрока. Это значит, что игрок видит свои ачивки во всех кампаниях этого бота, даже если играет разными персонажами.

### Что умеет мастер

Во вкладке **Достижения** мастер может:

- создать достижение;
- указать иконку-эмодзи;
- указать название и описание;
- указать тэг/раздел, например название кампании;
- выбрать награду-рамку из библиотеки уникальных рамок;
- выдать достижение привязанному игроку;
- добавить комментарий мастера при выдаче.

### Что видит игрок

У игрока появилась вкладка **Достижения**. Ачивки группируются по тэгам. По умолчанию игрок видит иконки, а при нажатии открывает полное описание, комментарий мастера, дату получения и награду, если она была.

### Кастомизация

В настройках персонажа кастомизация разделена на два блока:

- **Базовая кастомизация** — доступна всем сразу.
- **Уникальная кастомизация** — открывается только достижениями.

Запрос уникальной кастомизации у мастера больше не используется: теперь рамки выдаются через ачивки.

### Где всё хранится

Все достижения, выдачи достижений и открытые рамки хранятся локально в SQLite-базе:

```text
data/dnd_bot.sqlite3
```

Если используются аватарки, не забывай также сохранять папку:

```text
data/uploads/
```

### Как добавлять новые рамки в будущем

Рамки описаны в `app/db.py` в списке `COSMETIC_LIBRARY`, а визуальный стиль находится в `frontend/styles.css`. Чтобы добавить новую рамку, нужно:

1. Добавить запись в `COSMETIC_LIBRARY` с уникальным `id`.
2. Добавить CSS-класс рамки в `frontend/styles.css`.
3. Перезапустить проект.

После этого новая рамка появится в списке наград при создании достижения.

## v16 — модульная кастомизация: рамка + эффект

В этой версии кастомизация разделена на два независимых слоя:

- **Рамка** — форма и основной стиль вокруг аватарки.
- **Эффект рамки** — анимация, частицы, глитч, сканирование, пульсация или свечение.

Игрок может свободно комбинировать **любую открытую рамку** с **любым открытым эффектом**.

### Где это находится у игрока

Открой Mini App → вкладка **Настройки** → блок кастомизации:

1. **Базовая кастомизация** — обычные рамки, доступны всем.
2. **Уникальные рамки** — открываются достижениями.
3. **Эффекты рамок** — выбираются отдельно от рамок. Часть доступна всем, уникальные открываются достижениями.

### Как мастер выдаёт рамку или эффект через достижение

Открой Mini App → вкладка **Достижения** → **Создать достижение**.

У достижения теперь есть два независимых поля награды:

- **Награда-рамка** — можно оставить пустой.
- **Награда-эффект** — можно оставить пустой.

Ачивка может выдать:

- только рамку;
- только эффект;
- рамку и эффект одновременно;
- ничего визуального, если это просто памятная ачивка.

После выдачи достижения награда открывается для Telegram ID игрока и доступна во всех его кампаниях в этом боте.

### Какие эффекты уже есть

Базовые:

- Мягкое свечение
- Тихие искры
- Холодное дыхание
- Тёмная аура
- Неоновый пульс

Уникальные:

- Золотые искры
- Красный захват цели
- Цифровой глитч
- Свет софитов
- Пространственный сбой
- Огненная аура
- Разряд молнии
- Кровавый туман

### Где всё хранится

Всё хранится локально на твоём ПК:

- база: `data/dnd_bot.sqlite3`
- аватарки: `data/uploads/avatars/`

Новые таблицы для эффектов создаются автоматически при запуске проекта поверх старой базы.

## v18 — правки навигации, dev-тест и достижения

- Нижняя панель навигации поднята выше и учитывает safe-area на iPhone.
- Горизонтальная позиция нижней панели сохраняется при переключении разделов.
- В настройках мастера больше нет лишнего блока “Достижения и рамки”.
- В библиотеке достижений можно удалить достижение через его карточку.
- Для тестирования с одного устройства используется `DEV_MODE` в `.env`:
  - `DEV_MODE=1` — dev-сборка, включена по умолчанию;
  - `DEV_MODE=0` — prod-сборка, тестовое переключение отключено.
- В dev-сборке мастер может открыть настройки кампании и переключить вид на любого персонажа, чтобы проверить интерфейс игрока.

## v24 — тэги, искры и магазин

Добавлена аккаунтная система кастомизации:

- тэги рядом с именем персонажа;
- валюта аккаунта `✦ Искры`;
- магазин рамок, эффектов и тэгов;
- уникальную косметику купить нельзя — она открывается достижениями;
- достижения могут выдавать рамку, эффект, тэг и количество искр;
- мастер может вручную выдать игроку искры в разделе достижений;
- всё хранится локально в SQLite на вашем ПК.

Цены по умолчанию:

- обычная: 25 искр;
- редкая: 75 искр;
- эпическая: 200 искр;
- легендарная: 500 искр;
- уникальная: не продаётся.

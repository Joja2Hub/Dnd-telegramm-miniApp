# DND Local Host

Отдельный локальный сайт для игры по Wi-Fi без Telegram Mini App.

Все файлы запуска и локального сайта лежат в этой папке.

## Запуск

Дважды нажми:

```bat
local_host\START_LOCAL_SITE.bat
```

Сервер откроет:

```text
http://localhost:8000/local/
```

Игрокам в той же Wi-Fi сети дай адрес вида:

```text
http://192.168.0.104:8000/local/
```

Точный Wi-Fi адрес печатается в консоли при запуске. Используй адрес `192.168.x.x`, а не VPN-адреса вроде `10.x.x.x`, `25.x.x.x` или `26.x.x.x`.

## Важные батники

```bat
START_LOCAL_SITE.bat
```

Перезапускает локальный сайт на порте `8000`.

```bat
ALLOW_FIREWALL_PORT_8000_AS_ADMIN.bat
```

Один раз добавляет правило Windows Firewall для входящих подключений на порт `8000`.

```bat
KILL_PORT_8000_AS_ADMIN.bat
```

Принудительно убивает процесс на порте `8000`, если обычный запуск не смог его остановить.

## Как пользоваться

1. Открой `/local/` на компьютере мастера.
2. Выбери активную кампанию.
3. Нажми `Войти как мастер`.
4. Игроки открывают `/local/` со своих телефонов.
5. Каждый игрок выбирает своего персонажа.

## Как это устроено

`run_local.py` поднимает отдельный FastAPI-сервер-обертку:

- `/local/` и `/local-api/*` обслуживаются этой папкой;
- основной Mini App подключается как вложенное приложение;
- база остается общей: `data/dnd_bot.sqlite3`;
- локальный state хранится в `local_host/local_host_state.json` и игнорируется git.

Основной бот и локальный сайт лучше не запускать одновременно, чтобы они не спорили за SQLite.

Если видишь `database is locked`, закрой лишние окна бота/сайта. Если порт `8000` не освобождается, запусти `KILL_PORT_8000_AS_ADMIN.bat`, затем снова `START_LOCAL_SITE.bat`.
